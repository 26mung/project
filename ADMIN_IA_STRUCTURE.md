# 관리자 페이지 IA (Information Architecture) 구조

## 🎯 설계 원칙

1. **직관성**: 메뉴 이름만 보고도 기능을 알 수 있어야 함
2. **효율성**: 자주 사용하는 기능은 최상단 배치
3. **확장성**: 향후 추가 기능을 쉽게 넣을 수 있는 구조
4. **일관성**: /app 페이지와 동일한 디자인 톤앤매너

## 📊 메뉴 구조 (3단계)

### 1️⃣ 대시보드 (Dashboard)
**목적**: 시스템 전체 현황 한눈에 파악

#### 주요 지표 (KPI Cards)
- 총 사용자 수 (증감률)
- 총 프로젝트 수 (증감률)
- 오늘 가입자 수
- 활성 사용자 수 (최근 7일)
- 관리자 수
- 최근 30일 프로젝트 생성 추이

#### 차트/그래프
- 사용자 가입 추이 (라인 차트)
- 프로젝트 생성 추이 (바 차트)
- 프로젝트 상태 분포 (파이 차트)

#### 최근 활동
- 최근 가입한 사용자 (5명)
- 최근 생성된 프로젝트 (5개)
- 최근 관리자 활동 로그 (5개)

---

### 2️⃣ 사용자 관리 (User Management)
**목적**: 전체 회원 관리 및 권한 제어

#### 기능
- **사용자 목록**
  - 검색 (이메일, 이름)
  - 필터 (역할별, 가입일, 최근 로그인)
  - 정렬 (가입일, 이름, 이메일)
  - 페이지네이션

- **사용자 상세**
  - 기본 정보 (이메일, 이름, 생년월일)
  - 가입일, 최근 로그인
  - 현재 역할 (배지 표시)
  - 소유 프로젝트 수
  - 활동 이력

- **사용자 관리 작업**
  - 역할 부여/회수 (드롭다운)
  - 사용자 비활성화/활성화
  - 사용자 삭제 (확인 다이얼로그)
  - 비밀번호 재설정 링크 발송

- **벌크 작업**
  - 선택한 사용자 일괄 삭제
  - 선택한 사용자 일괄 역할 변경

---

### 3️⃣ 프로젝트 관리 (Project Management)
**목적**: 전체 프로젝트 모니터링 및 관리

#### 기능
- **프로젝트 목록**
  - 검색 (프로젝트명, 생성자 이름)
  - 필터 (상태, 생성자, 생성일)
  - 정렬 (생성일, 업데이트일, 이름)
  - 페이지네이션

- **프로젝트 카드 정보**
  - 프로젝트명
  - 생성자 이름
  - 상태 (Draft, In Progress, Completed)
  - 생성일/수정일
  - 요건 수
  - 완성도 (%)

- **프로젝트 상세**
  - 기본 정보
  - 생성자 정보
  - 요건 목록
  - PRD 문서
  - 활동 이력

- **프로젝트 관리 작업**
  - 프로젝트 삭제
  - 소유권 이전
  - 상태 변경

---

### 4️⃣ 역할 및 권한 (Roles & Permissions)
**목적**: 시스템 역할 및 권한 체계 관리

#### 기능
- **역할 목록**
  - Super Admin (최고관리자)
  - Admin (관리자)
  - User (일반 사용자)
  - Viewer (읽기 전용)

- **역할 상세**
  - 역할명/설명
  - 권한 레벨 (숫자)
  - 부여된 사용자 수
  - 권한 목록

- **권한 매트릭스**
  - 역할별 권한 표 (체크박스)
  - 프로젝트 CRUD
  - 사용자 관리
  - 역할 관리
  - 시스템 설정

- **역할 관리 작업**
  - 새 역할 생성
  - 역할 수정
  - 역할 삭제 (사용자 재할당 필요)

---

### 5️⃣ 시스템 설정 (System Settings)
**목적**: 전체 시스템 환경 설정

#### 기능
- **일반 설정**
  - 시스템 이름
  - 로고 업로드
  - 기본 언어
  - 시간대 설정

- **이메일 설정**
  - SMTP 서버 설정
  - 발신 이메일 주소
  - 이메일 템플릿 관리

- **보안 설정**
  - 세션 만료 시간
  - 비밀번호 정책 (최소 길이, 복잡도)
  - 2FA 활성화
  - IP 화이트리스트

- **데이터베이스**
  - 백업 관리
  - 백업 실행
  - 복원

---

### 6️⃣ 활동 로그 (Activity Logs)
**목적**: 시스템 활동 추적 및 감사

#### 기능
- **로그 목록**
  - 검색 (사용자, 액션)
  - 필터 (날짜, 액션 타입, 사용자)
  - 정렬 (시간순)
  - 페이지네이션

- **로그 항목 정보**
  - 타임스탬프
  - 사용자 (이메일/이름)
  - 액션 (로그인, 프로젝트 생성, 사용자 삭제 등)
  - IP 주소
  - User-Agent
  - 상세 내용

- **로그 타입**
  - 인증 (로그인, 로그아웃)
  - 사용자 관리 (생성, 수정, 삭제)
  - 프로젝트 관리 (CRUD)
  - 역할 관리 (부여, 회수)
  - 시스템 설정 변경

## 🎨 UI 컴포넌트

### 사이드바 (Sidebar)
```
플랫폼기획팀 (로고 + 타이틀)
├─ 대시보드 (홈 아이콘)
├─ 사용자 관리 (유저 아이콘)
├─ 프로젝트 관리 (폴더 아이콘)
├─ 역할 및 권한 (쉴드 아이콘)
├─ 시스템 설정 (톱니바퀴 아이콘)
└─ 활동 로그 (리스트 아이콘)

[하단]
├─ 메인 페이지로 돌아가기
└─ 로그아웃
```

### 상단 헤더 (Top Bar)
```
[페이지 제목] | [검색바] | [프로필 드롭다운]
```

### 메인 컨텐츠 영역
```
[페이지 헤더]
  - 제목
  - 액션 버튼 (추가, 새로고침 등)

[필터/검색 영역]
  - 검색 입력
  - 필터 드롭다운
  - 정렬 옵션

[컨텐츠 영역]
  - 카드/테이블 형태
  - 페이지네이션
```

---

## 📱 반응형 디자인

### Desktop (> 1024px)
- 사이드바 고정 (280px)
- 메인 컨텐츠 영역 유동적

### Tablet (768px ~ 1024px)
- 사이드바 축소 (아이콘만)
- hover 시 펼쳐짐

### Mobile (< 768px)
- 사이드바 숨김 (햄버거 메뉴)
- 모바일 최적화 레이아웃

---

## 🎯 MVP 우선순위

### Phase 1 (필수)
1. ✅ 레이아웃 구조 (사이드바 + 메인)
2. ✅ 대시보드 (기본 통계)
3. ✅ 사용자 관리 (목록, 삭제, 역할 부여)

### Phase 2 (중요)
4. 프로젝트 관리 (전체 조회)
5. 역할 및 권한 (읽기 전용)

### Phase 3 (향후)
6. 시스템 설정
7. 활동 로그
